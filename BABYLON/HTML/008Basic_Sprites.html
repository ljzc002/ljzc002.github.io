精灵


内容列表
  • 怎么做 ?
  • 下一步

在这个教程中，我们将学习怎样操作精灵。精灵是2D的图片或者动画，并且我们将使用它们来表现一个具有透明度通道的图像。精灵永远朝着相机。

今天，精灵经常被用来表现动画角色，和小微粒，还有模拟3D复杂物体比如树。（再比如FPS游戏中的准星，大型RTS游戏拉远视角时用贴图替代模型节省资源等）

怎么做 ?

1- 精灵管理器

如果你想要使用精灵，你需要建立一个“精灵管理器”通过将一个精灵的多个实例进行集合来优化GPU资源。这个管理器是必须的，即使你只想建立一个精灵。你必须这样写：

// Create a sprite manager
    var spriteManagerTrees = new BABYLON.SpriteManager("treesManagr", "Assets/Palm-arecaceae.png", 2000, 800, scene);

在建立一个精灵管理器时，你必须确定几个参数:

  • 名称: 这个管理器的一个名字.
  • 这个2D图片的URL（多数情况下，你会愿意使用一个具有透明度通道的图片格式，比如PNG）
  • 管理器的容量: 这个管理器中实例的最大数量（在这里例子里，我们将建立2000个树的实例）。
  • 每个小单元的大小，给你的图片设置一样的大小，像我们将在下面看到的一样.
  • 现有的场景，我们将把这个管理器加到这个场景里.

给出另一个例子，来看这个小人:

var spriteManagerPlayer = new BABYLON.SpriteManager("playerManagr","Assets/Player.png", 2, 64, scene);

这一次，我们只想要两个实例，并且我们把小人的尺寸设置为64.这里是我们的图片看起来的样子：

每一个小人的图片必须包括一个64像素的空间，不多不少.

2- 建立一个实例

既然我们有了管理器，我们可以建立与这个管理器连接的精灵的示例。建立一个实例就这样简单：

var player = new BABYLON.Sprite("player", spriteManagerPlayer);

在这里, 你让你的精灵显示了出来!

如果你想要给这个示例增加属性，你可以像操作其他网格一样操作它:

player.position.y = -0.3;

但是因为它是一个精灵，你可能需要使用特殊的属性：你可以修改他们的尺寸，或者是他们的方向：

player.size = 0.3;
    player.angle = Math.PI/4;
    player.invertU = -1;

从Babylonjsv2.1开始，你可以定义精灵的宽和高:

player.width = 0.3;
    player.height = 0.4;

你也可以保持使用player.size属性在这种情况下宽和高是一样的.

3- 精灵动画

精灵的一个优势是动画。你只需要载入一个包含所有动画图片的大图片文件，一个接一个。只需要注意遵守这个正方形的像素数符合你在管理器中的设置（比如64像素）。

这是一个完整的精灵图片看起来的样子:

这将会是你的角色在超过40种状态间变换，取决于其处境（行走，跳跃。。。）。Babylonjs的引擎可以自动读取多与一行的精灵，引擎为你服务:)

如果你想启动这个动画，简单的调用这个方法:

player.playAnimation(0, 43, true, 100);

通过这些参数调用« playAnimation » ,我们的角色将会在0帧到43帧之间进行动画变换。第三个参数指明这个动画是否会循环。最后一个参数是帧之间的延迟（这个参数越小，动画越快）。

最后，如果你想要显示其中的某一幅图片（比如最后一个，此时角色没有运动）只需调用:

player.cellIndex = 44;

你可以测试这个教程中的场景和代码。。。通过访问Babylonjs的 Playground精灵示例.

下一步

不要犹豫是否在你的场景中使用精灵：它们不会使资源紧张，并且它们在帮助你的应用程序保持高帧率时真的很有用

现在你知道了怎样建立一个简单却完整的动态场景，将会变得重要去讨论碰撞.
